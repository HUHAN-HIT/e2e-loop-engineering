## 3. 按严重性排序的发现

### 3.1 Blocker

#### B1 / RC2 — §0.4「自然隔离」是事实性幻觉

- **位置：** §0.4（第 ~40 行）
- **证据：** §0.4 措辞为「Claude Code 子智能体在隔离的上下文窗口中运行，**自然隔离写权限**」。「自然隔离」是事实主张。实际 Claude Code 子智能体**继承父级会话的工具配置**，除非显式声明 `tools:` 白名单或 `permission_mode`。
- **影响：** 如果工作节点具备 Write/Edit/Bash 权限，可绕过 §0.5 TD-001 直接写 `passing-tests.json`，使受信收集假设失效。
- **修复：** 显式声明工作节点子智能体的 `tools` 白名单（不含针对 `passing-tests.json` 路径的 Write/Edit/Bash），将「自然隔离」改为「显式隔离（通过 tools 字段配置）」。

#### B2' / RC1 — 一审报告的章节引用是系统性幻觉

- **位置：** `loop-engineering-adversarial-review-report.md` 全文
- **证据：** 报告引用 §12.7（第 49/61/75 行）、§15.1（第 62 行）、§16.3（多行）、§22（第 66 行），但文档真实结构为 §0–§17。
- **影响：** 一审报告的结论（裁决 `revise`）的*推理路径*无效；尽管裁决本身在新理由下仍成立。
- **修复：** 整体重写一审报告，改用文档真实章节。

## 9. 方法论说明（Mode A，Full 规模）

本报告由 5 名独立子智能体（Pro、Con、Correctness-Reviewer、Assumption-Reviewer、Risk-Reviewer）在隔离上下文中并行生成 → Cross-Examiner 在主上下文中比较声明 → Arbiter 裁决 → Scribe 渲染。

**关键方法论校正：** 上轮「元审查 M1–M5」是在单上下文中生成的（实际等同于 Mode D 降级模式），未与其他审查员独立对照，因此产生了幻觉章节引用。本轮通过 Mode A 的真独立性避免了这一陷阱——5 名独立审查员在「文档真实结构」上迅速收敛，纠正了上轮错误。
